문제
수빈이는 TV를 보고 있다. 수빈이는 채널을 돌리려고 했지만, 
버튼을 너무 세게 누르는 바람에, 일부 숫자 버튼이 고장났다.

리모컨에는 버튼이 0부터 9까지 숫자, +와 -가 있다. 
+를 누르면 현재 보고있는 채널에서 +1된 채널로 이동하고, 
-를 누르면 -1된 채널로 이동한다. 
채널 0에서 -를 누른 경우에는 채널이 변하지 않고, 
채널은 무한대 만큼 있다.

수빈이가 지금 이동하려고 하는 채널은 N이다. 
어떤 버튼이 고장났는지 주어졌을 때, 
채널 N으로 이동하기 위해서 버튼을 최소 몇 번 눌러야하는지 구하는 프로그램을 작성하시오. 

수빈이가 지금 보고 있는 채널은 100번이다.

입력
첫째 줄에 수빈이가 이동하려고 하는 채널 N (0 ≤ N ≤ 500,000)이 주어진다.  둘째 줄에는 고장난 버튼의 개수 M (0 ≤ M ≤ 10)이 주어진다. 고장난 버튼이 있는 경우에는 셋째 줄에는 고장난 버튼이 주어지며, 같은 버튼이 여러 번 주어지는 경우는 없다.

출력
첫째 줄에 채널 N으로 이동하기 위해 버튼을 최소 몇 번 눌러야 하는지를 출력한다.


#문제 이해
시작채널은 100번 
리모콘의 일부 버튼이 고장났다
채널은 0부터 무한대 
0에서 '-'해도 0
목표 채널까지 갈려면 최소 몇번을 눌러야 하는지 리턴


#문제 재정의
목표채널 가기 위해 쓸 수 있는 리모컨 버튼만을 이용해서 버튼을 누르는 최소 횟수





#방법계획
쓸수는 번호 버튼 배열을 구성

시작채널과 목표채널간의 차이 측정 

100-2 <= target <= 100+2
이면 증감으로 처리하면 끝

100,99,88
이외에는 번호를 눌러서 처리

쓸수 있는 번호 가장 큰 자릿수부터 누른다.
최대한 target과 가깝게 맞춘다
- 가깝게 맞추는 것의 기준



target과 일치하면 
일치하는데 쓰인 버튼 횟수 반환
아니라면 
차이 = target - 조합된 가장 가까운 채널 
채널조합에 소요된 횟수 와 '차이'를 더해서 return




#검증
0번에서 -해도 0이므로 상관없음
어차피 채널은 무한대이므로
